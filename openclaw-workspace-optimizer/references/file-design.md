# workspace 文件设计原则

## AGENTS.md

**本质**：岗位职责说明书。回答"这个 Agent 叫什么、做什么、怎么做、什么不做"。

**写好的三个要点**：

1. **写清楚边界，不只写"做什么"**  
   LLM 默认会"发挥创意"，边界比能力描述更重要。  
   ✅ "不要主动帮用户修改代码，除非用户明确要求"  
   ❌ "帮助用户解决代码问题"

2. **场景触发优于通用指令**  
   ✅ "当用户问技术问题时，用专业准确的措辞；当用户随意聊天时，语气可以轻松"  
   ❌ "始终保持专业语气"

3. **300-500 字是甜点区**  
   超过这个范围，重点被冲淡的概率急剧上升。

---

## SOUL.md

**本质**：性格档案，第一人称叙事，像人物小传，不像岗位说明书。

**四个组成部分**：
1. 自我叙事（我是什么样的存在）
2. 沟通风格（怎么说话）
3. 价值观和边界（什么是我不会做的）
4. 有趣的细节（可选，让 Agent 有稳定感觉）

**核心价值**：一致性建立信任感，信任感让用户愿意给它复杂任务。

---

## MEMORY.md

**只放这些**：
- 铁律（反复踩的坑，不能再犯）
- 高价值发现（非显而易见的环境约束）
- 当前仍有效的关键决策

**不放这些**：
- 原始对话摘要
- 单次任务的过程记录
- skill 文档里已有的内容
- 两个月前的项目背景

**大小上限**：10,000 chars。超了就提炼进 skill，然后从 MEMORY.md 删除。

---

## memory/ 目录

- `memory/YYYY-MM-DD.md`：按天滚动的工作记忆（原始）
- `MEMORY.md`：从日志里提炼出来的长期知识（精炼版）

两者关系：日志是原材料，MEMORY.md 是精炼品。  
定期把日志里的高价值内容提炼进 MEMORY.md，然后归档或删除旧日志。

---

## BOOTSTRAP.md

一次性引导文件，完成初始化后应该删除。  
文件还在 = workspace 可能还是"刚搭好"的状态。

引导流程：
1. 和用户聊几句，确定 Agent 名字、性格、emoji
2. 写进 IDENTITY.md
3. 记录用户基本信息到 USER.md
4. 一起完善 SOUL.md
5. 删除 BOOTSTRAP.md 本身
